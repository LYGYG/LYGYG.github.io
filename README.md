## 1，如何处理系统异常导致的数据异常?
这是一个系统设计问题了，单点技术不能解决系统问题。首先可以对数据分析，数据如何产生，如何组织，哪些部分重要，哪些部分不重要。数据在原始数据层能否冗余设计，保证关键数据有不同的通道可以达到消费端。再说各节点上。数据产生端要做风险估计，如果后端不能保证接收，产生端能缓存多长时间，如果有缓存延迟能否追回来。如果要丢数据，哪些数据可以丢，哪些数据不能丢。数据传输过程要考虑同级备份，或者考虑前置一部分数据过滤和清洗，进行数据分流，减少前后数据的依赖，关键信息要有冗余。存储端要进行数据清洗，修正。需要不同路径来的数据做校正，还需要一些数据模型来做数据填充和插值。监控主要是重启和告警，由它影响到数据就不好了，分析数据时很难区分哪一部分是因为监控导致的数据不准。

## 3，客户临时变更需求怎么办？
 1. 深入了解变更需求：与提出人沟通，了解变更需求的具体背景和原因，重新评估变更的重要性和优先级，以及对项目进度和预算的影响。
    
 2. 分析变更成本：评估变更对研发进程和资源的影响，例如可能需要重新规划开发流程，增加人力、延长开发周期或添加新的技术要求等。
 
 3. 更新产品文档：及时更新产品需求文档和其他文档，确保所有相关人员都能得到更新的信息。
 
 4. 与团队成员协商：与团队成员密切协调，讨论变更如何影响项目进度和质量，并根据需要进行适当的调整。
 
 5. 保持沟通和协作：与产品经理保持良好的沟通和协作，及时反馈和解决问题，确保研发进程和产品质量不受影响。
 ## 4，你是根据什么对需求进行排序？
  需求优先级需要遵循哪些原则呢？
一、Kano模型（必备型需求>期望型需求>兴奋型需求>无差别需求>反向需求）
Kano模型是东京理工大学教授狩野纪昭（Noriaki Kano）发明的对用户需求分类和优先排序的工具，分析用户需求对用户满意的影响为基础，体现了产品性能和用户满意之间的非线性关系。

二、重要性/紧急性
《麦肯锡方法》针对时间管理提出了一个高效的管理方法-四象限法，主要解决“如何合理安排事情处理的顺序”。纵轴以重要性为单位，横轴以紧急性为单位，两两匹配。重要性：是否是产品的核心功能，是否和商业利益相关，是否在长远上看是当前就要打好地基避免将来推倒重来的。紧急性：当下要完成的，且离deadline很短，就是紧急性高的，相对有充足时间的，紧急性就没那么高。

三，开发量
优先见效快且开发量不大的，快速迭代。
最后做开发量很大且见效慢的，这可能是未来的机会。
## 5，如何进行产品规划
 第一步，明确做规划的目的

 面对不同汇报人群：向上汇报：重点应该是突出规划的方向和你最终期望的效果，向领导说明怎么得出的方向，为什么可以这么做。向项目组宣讲：重点应该是总结和突出规划细节，告诉项目组他们之前完成的成绩和接下来要做什么。

 不同产品阶段：初步/成长期的产品：做好方向上的判断，也就是分析趋势上，避免走错方向；快速成长：突出产品路线图，指导迭代版本；稳定期：可以增加数据分析和优化方案，更多的用户分析和竞品分析。

 不同的工作时间点：年终、季终会有各种总结和规划，规划是很好的对过去总结和回望，对下一年做出大体的规划进入工作高速期，会有不断的版本迭代，这时候应该突出每项的规划细节，包括需求List，优先级等等，而这种形式的不一定是要写PPT，用execl表格就能搞定；

 第二步，列提纲
 
 数据总结和分析：过去项目的关系数据情况，分析下趋势，得出产品弱点和不足。行业趋势和竞品分析：产品所处的行业趋势怎么样，出现了哪些方向。产品定位：趋势有了，那么产品本身在这个行业环境的身份是怎么，结合身份、环境趋势、用户需求，你希望做什么，有哪些方向和Key Point，也就是产品定位。（最好用一句关键句表达出来，比如每个产品都有的slogan）产品规划：结合定位中提出的Key Point，提出完成KeyPoint需要解决哪些问题，提出解决方案（关键句总结）产品路线图（RoadMap）：完成规划需要做哪些功能，在什么时间段完成什么功能。目标：最终期望达到的目标；

 第三步，往提纲里填内容

 1.数据总结和分析
将项目的关键指标用图表的形式表达出来，关键指标大致分几类：流量、转化率、粘性、营收。流量包括：用户数总量，新增用户比率等等；转化率包括渗透率、点击率等等（当然每个行业都有不一样的转化率定义，比如ROI）；粘性包括留存率、人均启动次数、人均使用时长等等。针对关键指标的变化趋势，分析产品优劣势。（数据分析就不展开讲了）；

2.行业趋势&竞品分析
这一部分需要准备的前期工作比较多，也看平时积累的多少，比如平时写的竞品分析等等，都可以用上了。行业趋势报告各大平台也会有，友盟、智观、企鹅智库、TalkingDate等等。请用几句关键句总结下你认为的行业趋势，并分别用一页PPT来阐述这个趋势。（灵活阐述）；

3.产品定位
产品定位应该结合产品所处的环境，也就是第二部分的行业趋势，加上产品本身的身份和用户需求来得出，我称之为Key Point；

4.产品规划
针对提出的Key Point，一个个阐述。大致的逻辑是，列出你期望达到的目标，阻碍到达目标的的问题是什么（产品问题和现状），提出解决方案。

5.产品路线图
规划完了之后，会有一堆要做的事情，也就是需求list，将这些需求list大致按时间线排列起来，当然也可以按里程碑串联起来。我这里列的是时间线的路线图；

6.目标
提出合理的目标，可以是产品目标也可以是KPI（一般来说KPI是在规划前提前测算好的），这部分也可以说是打鸡血部分，对项目组有鼓励作用，对领导可以作为争取资源的依据。
## 6，如何进行产品年度规划
 一、公司战略分析
老板让你做这件事情的目的是什么，与领导充分沟通，了解公司对该产品的希望，从而认清产品未来之路。这时候，你要清楚：
1、公司的战略布局是如何？2、公司会分配哪些资源给你？3、公司KPI是什么？

二、行业分析
通过行业状况的了解、竞争对手的分析，清楚自己产品目前所在的位置和阶段。通过这个环节的了解，你需要得出的是：
产品的市场影响率如何？产品的用户口碑如何？行业的发展趋势如何？一份完整的竞品分析文档；

三、产品核心定位分析通过了解老板的意图、行业分析、竞争对手的分析，你需要拿出一份像样的自身产品的核心定位，这时候你要注意：1、该产品在整个公司的产品线中是充当什么样的角色2、和竞争对手的核心差异是什么3、满足用户的核心需求是什么4、用户为什么要用你的产品，而不是其他的产品；

四、这个产品的核心功能有哪些？
知道了意图，现状，产品的定位，就要基于以上需求设计产品的功能了，这时候你要注意：
1、功能的优先级2、功能的重要程度

五、确定这个项目可以投入的资源
包括人力、市场预算等

六、开始制定计划
很多时候老板最关注的反而是这点，他们要你做产品规划的目的是两个原因，1是希望提炼核心产品， 2是系统有一个对未来的计划，他们怕人闲着。

## 8，产品经理的变迁

   分为三个时代，最早的是快消品的产品经理，即使是同类品牌消费者购买的动机也是不一样的，针对消费者满足不同的需求。聚焦用户需求，调配资源去满足用户需求；
   无论什么时代，产品经理的核心还是发现需求、满足需求和超越需求。时刻站在用户身边嫁接用户需求和产品开发之间的桥梁。
   职业跟着时代走，只要底层逻辑和需求还在，这个工种就会以新的形态出现在新的市场中。
   产品经理应该具备服务意识，用户是谁用户的痛点是什么，必须创造出能让用户感觉舒服的产品特点，以人为本。最牛逼的产品经理应该是能够创造需求。

## 9，理想ONE汽车产品
   
   从不被理解到被市场验证，特别不被看好的地方是使用了“增程技术”，车不是给投资人做的，“除去一切不可能，剩下的再不可能也是真相”。纯电车是新能源主流，但是充电桩设备不足，是当下面临的普遍问题。燃油车出行是“城市用油、长途用油，不够经济”、纯电车出行是“城市用电、长途用电，具有纯电车的里程焦虑”。而从感性的角度出发，除了考虑家庭使用场景，以及给每个家庭成员良好的体验外，最重要的是让一家人出行有心理上的安全感，带着家人开长途，有极大可能是出去游玩，如果在这个时候找充电桩等充电的状况，不仅体验很差，还容易让乘坐者产生焦虑，破坏一次本来美好的游玩体验。理想one创造全国首创“城市用电，长途发电”的全新电动模式，就成了一个很合理的追求，理想one使用“增程技术”也就很正常了，在当下实际情况中，增程是更适合的解决方案；
   四屏交互技术采用T型的屏幕排布，三连横屏为“数字仪表屏、中控屏、副驾驶娱乐屏，下面为功能控制屏”，主要考虑到副驾驶体验的缺失，家庭用车副驾一般坐的是重要的家庭成员，使用率65%（现在用手机的人多，可以考虑副驾驶的投屏需求）。
   当一家人坐在车里，车不仅仅是交通工具，是一个提供全家人交流、娱乐、互动的空间，所以做全车语音交互。家庭成员的位置都同等重要，都对车辆有语音需求，还大大减少让司机操作的过程，理想one做了四音区语音交互方案，个人用户、商务用户和家庭用户就是三种不同思路，需求从场景中来，接送上下学车辆要快上快下，减少七座车的车门被座椅遮挡场景，所以有的采用六座而且过道够宽，可放儿童座椅；
